# Ticket Plans

Optional index for per-task planning notes.

Use it only if your workflow creates explicit plan documents before implementation.

## Suggested Status Values

| Status | Meaning |
| --- | --- |
| `draft` | Plan exists but is not approved |
| `approved` | Ready to implement |
| `in progress` | Work is underway |
| `done` | Work completed and verified |

## Notes

- One plan per task.
- Update the existing plan instead of creating duplicates.
- Keep the plan short and decision-oriented.
