# Architecture Decisions (ADR)

Use this file to record design decisions that future engineers or AI tools should not have to rediscover.

## Create An ADR When

- a new system or durable module boundary is introduced
- layer responsibilities change
- a contract or permission model changes across features
- a library or technical direction is chosen and will be costly to reverse

Do not create ADRs for routine feature work or small bug fixes.

## Template

### [ADR-ID] - Short Title

- **Date:** YYYY-MM-DD
- **Status:** `proposed` | `accepted` | `superseded` | `deprecated`
- **Context:** What problem led to this decision?
- **Decision:** What was chosen?
- **Alternatives considered:** What else was considered and why was it rejected?
- **Consequences:** What does this make easier or harder?
- **Related work:** task, PR, or incident references
- **Superseded by:** ADR-ID or `none`
