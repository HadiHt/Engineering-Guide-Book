# Development Workflow

Use this for normal task execution.

## Fresh Start

1. Check whether the work is already in progress.
2. Confirm the task is clear enough to implement.
3. Detect scope: `frontend`, `backend`, or `both`.
4. Load the required pages from the hub index.
5. Implement the smallest correct change set.
6. Review the diff and run relevant checks.
7. Update the task log or project memory if your team uses one.
8. Verify the Definition of Done.

## Resume Work

If work already exists on the branch or in the task log, follow **Session Resume Protocol** instead of treating it as a fresh start.

## Notes

- Prefer task-driven changes over broad cleanup.
- If a blocker spans multiple layers, surface it early.
- Treat review as part of implementation, not an afterthought.
