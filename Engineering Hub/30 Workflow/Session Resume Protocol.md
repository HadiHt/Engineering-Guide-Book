# Session Resume Protocol

Use this when work already exists from a prior session.

## When It Applies

- there are unfinished changes on the branch
- a PR already exists
- a task log or project memory entry says the work is in progress

## Resume Procedure

1. Read the existing task summary, log, or project memory if available.
2. Re-detect the scope from the actual code and task, not from memory.
3. Reload the required standards for this session.
4. Inspect the current diff and branch state.
5. Identify what is done, what is partial, and what is still missing.
6. Re-check any known risks or follow-ups.
7. Continue with the smallest next step.

## Rules

- Do not assume the previous session was correct.
- Do not start coding before reviewing the existing state.
- Do not create duplicate task-log entries if one already exists.
