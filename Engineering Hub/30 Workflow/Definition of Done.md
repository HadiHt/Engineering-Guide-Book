# Definition of Done

Use this checklist before declaring a task complete.

## Before Coding

- [ ] Scope is known: `frontend`, `backend`, or `both`
- [ ] Required standards were loaded
- [ ] Write boundaries are understood
- [ ] Any structural ambiguity was resolved

## Implementation Complete

- [ ] The requested behavior is implemented
- [ ] Changes stayed inside scope
- [ ] No protected or risky areas were changed without approval
- [ ] Touched code follows the relevant standards
- [ ] User-facing text, permissions, and data handling were reviewed where relevant

## Review Complete

- [ ] The diff was reviewed for regressions
- [ ] Tests were added or updated where needed
- [ ] Relevant local validation passed, or failure was reported clearly
- [ ] No unresolved blocking issues remain
- [ ] Residual risks and assumptions are documented

## Ready To Hand Off Or Merge

- [ ] Branch, commits, and PR hygiene meet team standards
- [ ] Project memory or task log was updated if this workflow uses one
- [ ] Any follow-up work is explicit
